Spring Boot 예외 처리

전역 예외 처리와 커스텀 예외 시스템 데모
웹 사용자 관리 REST API 테스트 404 오류 테스트 런타임 오류 테스트

예외 처리 테스트 방법

  • 404 오류: 존재하지 않는 사용자 ID 조회
  • 중복 오류: admin, root 등의 사용자명으로 가입
  • 검증 오류: 필수 필드를 비워두고 폼 제출
  • 비즈니스 오류: @blocked.com 도메인 이메일 사용
  • 시스템 오류: POST /api/users/test/system-error

API 테스트 명령어

# 사용자 생성 (검증 오류 유발)
curl -X POST http://localhost:8080/api/users \
  -H "Content-Type: application/json" \
  -d '{"username":"", "email":"invalid", "name":""}'

# 존재하지 않는 사용자 조회 (404 오류)
curl http://localhost:8080/api/users/999

# 시스템 오류 테스트
curl -X POST http://localhost:8080/api/users/test/system-error
